Golang, also called Go, was developed by Google in 2009. Those developers wanted a sexy, agile, effective language that would be simpler to read and work with than C and C++, which is what they were relying on at the time (to their dismay).

What is the ‚Äúfront-end‚Äù?

The frontend of the software is what the user sees. If you look at a website, even something as simple as a login page, everything you see on the page is considered the frontend. Frontend code dictates the style and layout of the page.

If you know anything about programming, you may already be starting to get an inkling about why Go is best for the backend. In case not, I‚Äôll spell it out here:

  • Go has built-in concurrency called ‚Äúgoroutines‚Äù. This means it can multitask, which is important for implementing high-performance servers and networked applications.

Go is used heavily by Google to support their back-end infrastructure. It‚Äôs good for back-end development because that‚Äôs what it was designed for.

Can Go be used on the front-end? üîó

I mean, if you really want to, you can use Go as a frontend language, the same way you can use a hammer to make a screw go in.

One way to use Go on the front-end is by compiling it to WebAssembly (Wasm). WebAssembly is a binary format that is supported by modern web browsers, and it allows developers to run code in the browser that was written in languages other than JavaScript. When it comes to writing front-end code, at least on the web, there are only two options:

  • JavaScript
  • Web Assembly (WASM)

Web Assembly is still new, but it does work. So, if you really want to write so front-end applications in Go, you can, but I‚Äôd argue you‚Äôre better off waiting on the tooling to mature a bit more.

In short, let go (pun intended) of those dreams and use Go in a context where it shines: as a command-line interface tool, and as a backend language.

What has Go been used for? üîó

Go is primarily a backend language, and has been used to build components in:

  • Docker. The Docker daemon, a system that manages containers, is built with Go.
  • Kubernetes. The Kubernetes control plane, which is the backend component that manages a cluster of servers, is built with Go.
  • Terraform. The Terraform backend, which is the component that manages and applies infrastructure changes, is written in Go.
  • InfluxDB. The InfluxDB server, which is a backend component that stores and retrieves data, is built with Go.

And of course, Go‚Äôs own progenitor Google has made use of Go to create or support:

  • Google App Engine: Google App Engine is a cloud computing platform that allows developers to build and deploy web applications and mobile backends. The App Engine runtime, which is in charge of executing applications and handling requests, is built with Go.
  • dl.google.com: dl.google.com is a web service that provides access to Google‚Äôs software downloads, including Chrome, Android Studio, and various other tools. The dl.google.com backend, which is responsible for serving download files and tracking download statistics, is served by Go.
  • gVisor: gVisor is an open-source security sandbox that allows developers to run untrusted code in a secure environment. gVisor is built with Go.
  • Google Cloud Functions: Google Cloud Functions is a serverless computing platform that allows developers to run code in response to events. The Cloud Functions runtime, which is responsible for executing functions and handling requests, is built with Go.
